# Homepage Blue Theme Shift Design

**Goal**

将首页当前偏荧光绿的主视觉高亮统一替换为偏清透的雾感科技蓝，同时保持现有金色点缀、布局结构与动效节奏不变，让页面更稳重、更统一，并继续贴近目标站的高级工业感。

## Scope

- 替换首页沉浸式头部里的亮绿高亮体系
- 统一导航当前激活色、语言切换激活态、Hero 主按钮、浮动徽标、亮点标签、局部光晕与深色面板底色
- 不调整 About、Footer、内容结构和文案
- 不移除现有金色辅助色，避免全站失去层次

## Approaches

### A. 直接把所有绿色字面量改成蓝色

优点是快，改动少。缺点是后续难继续微调，而且不同组件可能出现蓝色深浅不统一的问题。

### B. 抽取一组首页沉浸式蓝色变量并统一替换

优点是统一性最好，后续如果还要更靠近目标站，只要改少量变量即可；也能避免按钮、导航、高亮点蓝得不一致。缺点是这次会多做一步整理。

### C. 同时把全站暖金主按钮也一起改成蓝色

统一程度更强，但会把现有“蓝金工业”层次打平，整体反而更单一，也偏离当前已经调顺的稳重方向。

## Recommendation

采用 **B**。

原因是用户希望“全面改成蓝色，但不是深蓝色”，同时又明确要求“统一”。最稳妥的方式不是单点换色，而是建立一组浅中蓝变量，专门服务当前沉浸式首页主视觉，并保持暖金点缀继续负责辅助层级。

## Color Direction

- 主高亮蓝：偏雾感、偏科技感，不发紫、不发青
- 亮面蓝：用于主按钮与激活态
- 柔和蓝光：用于阴影、光晕、扫光和背景扩散
- 深底蓝灰：替换当前偏绿色的深底，保证页面气质仍然稳重

建议使用的蓝色范围：

- 主按钮渐变：`#9fd2ff -> #79baf3`
- 激活高亮：`#a9d9ff`
- 发光阴影：`rgba(121, 186, 243, 0.24)`
- 深色面板底：`#0f1c2a` / `#132438`
- 深色文字反差：`#102235`

## Affected Areas

- `src/styles/global.css`
  - 导航品牌与导航高亮
  - 语言切换激活态
  - 沉浸式 hero 背景光晕
  - Hero CTA 主按钮
  - Hero 小亮点、漂浮胶囊、右侧深色卡框
- `src/components/ui/button.tsx`
  - 如果需要与首页 CTA 的蓝色策略保持一致，调整通用按钮聚焦色或渐变引用方式

## Testing

- 新增一个样式测试，锁定“蓝替绿”的关键字符串
- 验证首页 Hero 相关样式中不再保留旧的荧光绿渐变与绿色深底
- 运行现有 Hero CTA 动效测试，确保换色不会破坏动效相关规则
