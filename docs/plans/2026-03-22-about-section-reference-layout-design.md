# 公司简介参考站布局设计

**日期：** 2026-03-22

**目标：** 将首页“公司简介”模块从当前重叠双卡形式，调整为更接近目标站的“左图右文”独立双栏结构，强化稳重、清晰、企业化的展示感。

---

## 现状问题

当前“公司简介”模块的问题不在内容，而在版式关系：

- 左侧图片卡与右侧内容卡同时存在位移。
- 右侧内容卡通过负边距压到左图上，视觉上产生碰撞。
- 左右卡片又分别带有上下位移，导致层次不是“错落”，而是“重叠打架”。

这套结构在内容稀疏时还能勉强成立，但当前文字量、统计块和图片角标一起出现后，会明显显得拥挤和不整齐。

## 参考站提取出的布局逻辑

参考站这一段的高级感主要来自这些点：

- 左侧是完整主视觉图片区块。
- 右侧是独立内容面板，不压住图片。
- 信息按照“标题/说明 -> 小数据 -> 条目列表”的顺序自然展开。
- 层次依赖分栏比例、留白、卡片材质和模块节奏，而不是靠两张大卡互相压住。

## 确认采用的方案

采用“左图右文、不重叠”的参考站逻辑：

- 左侧保留单一大图卡。
- 左图上的小角标保留，但尺寸和位置要更克制。
- 右侧内容卡不再使用负边距，不再横向压入图片。
- 右侧顶部先展示两枚小数据卡。
- 右侧下方保留条目列表，节奏更整齐。
- 整段只保留轻量 reveal 动效，不保留物理重叠错位。

## 结构调整

### 左侧

- 保留 `about-image-card` 作为主视觉容器。
- 保留图片右下角 badge，但视觉上更精简，避免抢正文注意力。

### 右侧

- `about-content-card` 改为正常文档流卡片，不再 `margin-left` 压住图片。
- 顶部的 `about-mini-stats` 保持两列，但要与正文产生更清晰的区隔。
- `about-points-grid` 继续保留，但条目整体要更像企业站信息块，不像功能列表组件。

## 视觉方向

- 图片和内容卡需要平齐或近似平齐，不再产生交叉遮挡。
- 外轮廓要统一成稳重的大圆角，减少“漂浮拼贴”感。
- 卡片阴影要收敛，避免左右两块都很重。
- 通过更明确的左右比例和上下间距，建立高级感。

## 风险与约束

- 不能破坏当前移动端布局回退。
- 不能影响首页其余模块的节奏。
- 不新增无关装饰，不引入新的复杂组件。
- 保持内容结构尽量稳定，优先调布局与样式。

## 验证标准

- 桌面端“公司简介”左右两块不再重叠。
- 左图右文关系清晰，整体更接近目标站逻辑。
- 右侧顶部小数据和条目列表层级清楚。
- 移动端回退后仍然自然。
- 首页相关测试与构建通过。
