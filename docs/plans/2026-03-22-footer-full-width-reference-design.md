# 底部全宽页脚参照目标站设计

**日期：** 2026-03-22

**目标：** 将当前首页底部从“中间一条的独立深色卡片”改为“整段全宽铺开的页脚区域”，参照目标站的 footer 组织逻辑，让背景横向铺满，内容在内部容器中排布，视觉上与页面主体自然衔接。

---

## 当前问题

当前底部虽然已经具备较完整的内容分栏，但整体视觉仍然像一张悬浮在页面底部中央的深色大卡片，原因主要在于：

- `site-footer__panel` 负责了承载背景、圆角、阴影和边框
- 外层仍然是受限宽度的居中容器
- 整个 footer 的深色区域没有真正铺满页面横向空间

这会让页面结尾看起来像“中间一条”，与用户要求和目标站的 footer 逻辑不一致。

## 目标站提炼出的底部结构逻辑

从目标站可提炼出几条关键规律：

- 页脚背景是整段全宽，而不是单独的一张卡片
- 内容依旧放在 `.container` 这样的内部宽度容器中
- footer 和上方模块之间通过深色背景、过渡层和留白建立衔接
- 内部是清晰的多列信息组织，而不是靠外轮廓卡片制造层次

换句话说，目标站的层次来自“全宽背景 + 内容容器 + 多列排版”，而不是“居中深色卡片”。

## 确认采用的方案

采用方案 A：

- 移除当前 `site-footer__panel` 这类独立卡片式视觉壳层
- footer 背景整段全宽铺开
- 内容容器仅负责控制最大宽度和列布局
- 通过顶部渐变过渡与轻纹理增强层次，但不再产生卡片悬浮感

## 结构调整

### 外层

- `footer.site-footer` 直接承担全宽背景职责
- 增加轻量背景装饰层或伪元素，用于与上一模块平滑过渡

### 内层

- `site-footer__inner` 保留为内容容器
- `site-footer__grid` 继续承担四列布局
- `site-footer__bottom` 保留版权信息，但边界线应更自然地嵌入整段页脚，而不是挂在卡片底部

## 视觉方向

- 整段 footer 使用深海军蓝到近黑的渐变底色
- 上边缘增加更柔和的过渡层，避免与上一个 section 生硬切断
- 去掉大圆角黑卡和重阴影
- 让 logo、标题、链接、联系信息在 full-width 背景中更舒展

整体气质应偏“稳重、完整、收束”，而不是“底部再放一张展示卡”。

## 风险与约束

- 不能让 footer 因为铺满全宽而变得过空或过散
- 不能破坏现有移动端单列回落
- 不能让顶部过渡过重，压住上方联系模块
- 现有 CMS 动态字段接线必须保留

## 验收标准

- 底部不再呈现“中间一条悬浮卡片”的观感
- footer 背景横向铺满页面
- 内容依旧整齐、稳重、清晰
- 四列内容在桌面端保持有序，移动端自然回落
- 测试和构建通过
