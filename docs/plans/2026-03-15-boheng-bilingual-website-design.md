# 大连博恒新技术有限公司官网设计文档

**日期：** 2026-03-15

**项目目标：** 基于参考站的单页展示节奏，建设一个面向工业制造客户的中日双语企业官网，并配套轻量后台用于新闻发布、页面内容维护、图片管理和留言查看。

## 1. 项目背景

当前项目目录为空，需要从零搭建。用户提供了参考站页面风格和公司资料文档，已明确以下核心需求：

- 官网内容以大连博恒新技术有限公司真实业务资料为基础
- 首页与整体视觉参考提供的单页模板，但不能沿用原有家装/屋面工程行业语义
- 网站需支持中文与日文完整切换
- 右上角提供语言切换入口
- 需要后台管理能力，用于发布新闻、管理前台内容、查看留言
- 数据量较小，希望方案轻量、易维护

## 2. 设计结论

### 2.1 技术架构

采用 `Astro + PocketBase` 的组合：

- `Astro` 负责前台企业官网渲染、双语路由、SEO 和页面性能
- `PocketBase` 负责轻量数据存储、后台管理界面、认证、文件上传和 API

选择原因：

- Astro 适合内容展示型官网，首屏加载快，SEO 友好
- PocketBase 底层使用 SQLite，适合小数据量企业站
- PocketBase 自带管理后台，避免从零开发复杂 CMS
- 前台与后台职责清晰，后续扩展新闻、案例、图片更方便

### 2.2 部署形态

- 前台采用 Astro SSR 模式运行，实时读取 PocketBase 数据
- 后台与数据服务由 PocketBase 进程提供
- 适合部署在可长期运行服务的云服务器或 Windows/Linux 主机

不采用纯静态导出，原因是：

- 新闻发布后需要立即生效
- 页面内容在后台修改后希望直接更新
- 留言提交需要服务端处理

## 5. 双语设计

### 5.1 路由策略

采用独立路径前缀：

- 中文：默认路径
- 日文：`/ja/` 前缀

优点：

- SEO 清晰
- 用户可直接收藏语言版本链接
- 便于为中日文分别设置标题、描述和 Open Graph 信息

### 5.2 内容管理策略

后台所有可展示内容均按双语字段维护：

- `title_zh` / `title_ja`
- `summary_zh` / `summary_ja`
- `content_zh` / `content_ja`

语言切换规则：

- 在首页切换时跳转到对应语言首页
- 在新闻详情切换时跳转到对应语言新闻详情
- 若某条内容缺失日文版本，则后台不允许发布至日文站

### 5.3 SEO

每个双语页面需要：

- 单独的 `title` 与 `meta description`
- `hreflang="zh-CN"` 与 `hreflang="ja-JP"`
- 规范化 URL
- 适合工业企业检索的关键词

## 7. 视觉与交互方向

参考站保留“单页大区块 + 大图 + 卡片 + 滚动动画”的节奏，但整体风格转为工业制造企业视觉：

- 主色建议为深蓝、钢铁灰、金属银
- 点缀色使用安全橙或科技青
- 字体偏稳重、技术感，不走家装温暖路线
- 图片应优先使用设备、车间、工艺、产品细节，而非家庭服务场景
- 页面动画控制在轻量范围，保持专业感

语言切换按钮放在导航右上角，桌面端显式显示，移动端放入折叠菜单顶部。

## 8. 数据流设计

前台读取流程：

1. Astro 页面按当前语言请求 PocketBase 内容
2. 服务端筛选已发布内容
3. 页面根据语言字段渲染相应文案

留言提交流程：

1. 用户在前台提交表单
2. Astro 服务端校验字段
3. 服务端写入 PocketBase `messages`
4. 前台返回成功或失败状态提示

新闻发布流程：

1. 管理员在 PocketBase 后台填写中日文内容
2. 设置封面、发布时间与发布状态
3. 前台新闻列表自动读取最新可见内容

## 9. 错误处理

- 新闻不存在时显示定制化 404 页面
- 接口异常时首页保留基础静态骨架和错误提示
- 留言提交失败时提示用户稍后重试，并保留已输入内容
- 缺失某语言内容时不在对应语言站点展示该内容
