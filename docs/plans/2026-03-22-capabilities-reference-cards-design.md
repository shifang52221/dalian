# Capabilities Reference Cards Design

**Goal**

把首页“核心能力”六张卡片从当前偏工程信息卡样式，改成接近参考图二的展示方式：顶部居中图标、下面大标题、下面说明文字，整体白底大圆角，悬停时整卡变为统一主题蓝纯色，不使用渐变 hover。

## Current Problem

- 现有卡片仍然是左上数字块 + 标题正文左对齐
- hover 使用深色渐变遮罩，和参考图二不一致
- 卡片重心偏上偏左，留白感不够干净
- 视觉语言更接近工程列表，不像参考图那种规整展示卡

## Reference Interpretation

用户给的参考图二核心特征是：

- 卡片为白底大圆角
- 顶部中心位置是图标容器
- 图标下方是标题
- 标题下方是正文
- 内容整体居中排布
- 鼠标移入后整卡切换为主题纯色，不加渐变，不加杂光

## Recommendation

采用单独的 capabilities 卡片变体，而不是直接重写所有 `service-card`。

这样可以：

- 只影响这一组能力卡
- 保持其他新闻、项目、优势卡片不被误伤
- 精准对照参考图逐项收样式

## Layout

- 卡片添加专属类，例如 `service-card--capability`
- 内部结构改为：
  - 顶部图标容器
  - 标题
  - 正文
- 整卡使用居中排版
- 六张卡片统一高度、内边距、圆角和间距

## Hover

- 整卡 hover 直接切换到主题蓝纯色背景
- 不使用 `linear-gradient` 作为 hover 主背景
- 标题、正文、图标统一切到高对比浅色
- 图标壳层保留轻微透明白，不做渐变

## Testing

- 新增一个能力卡片回归测试
- 验证组件包含专属 capability card 结构类
- 验证样式存在 centered layout 和纯色 hover
- 验证不再依赖能力卡片 hover 渐变遮罩
